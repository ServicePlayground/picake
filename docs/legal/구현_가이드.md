# 약관 동의 구현 가이드

코드베이스 분석 기준: 2025-05-21

---

## 개요: 약관 파일 구성

```
apps/web-user/public/legal/
  terms-of-service.html   ← 서비스 이용약관
  privacy-policy.html     ← 개인정보 처리방침
  location-terms.html     ← 위치정보 이용약관 (web-user 전용)

apps/web-seller/public/legal/
  terms-of-service.html   ← 서비스 이용약관 (동일 파일)
  privacy-policy.html     ← 개인정보 처리방침 (동일 파일)
```

Next.js와 Vite 모두 `public/` 폴더의 파일을 루트 경로로 정적 서빙합니다.
약관 링크는 `href="/legal/terms-of-service.html"` 형태로 연결하면 됩니다.

---

## 개인정보 제3자 제공 동의 — 별도 페이지 불필요

개인정보 보호법 제17조는 제3자 제공 동의를 개인정보처리방침 동의와 **분리된 체크박스**로 받도록 요구합니다.
단, 별도 페이지(HTML 파일)는 불필요하고 **가입 화면 내 인라인 고지 텍스트 + 별도 체크박스**로 충분합니다.
개인정보처리방침 제3조 링크로 연결하면 됩니다.

```tsx

{ label: "위치정보 이용약관", href: "/" },
```

`LinkListItem` 컴포넌트가 내부적으로 Next.js `<Link>`를 사용하므로,

약관 특성상 **새 탭**으로 여는 게 UX상 적합하면 `target="_blank"` 처리가 필요합니다.

`LinkListItem` 컴포넌트에 `external?: boolean` prop을 추가하거나,
또는 해당 세 항목만 `<a href="..." target="_blank">` 태그로 직접 렌더링하세요.

## 6. 백엔드 — 동의 기록 저장 (필수)

개인정보 보호법은 **동의 사실을 회사가 입증**해야 합니다. 반드시 DB에 기록해야 합니다.

### Prisma 스키마에 추가

```prisma
// apps/backend/src/infra/database/prisma/schema.prisma

model ConsentRecord {
  id                   String   @id @default(cuid())
  consumerId           String?
  sellerId             String?
  termsVersion         String
  privacyVersion       String
  termsOfService       Boolean
  privacyPolicy        Boolean
  thirdPartyProvision  Boolean  @default(false)
  locationService      Boolean  @default(false)
  marketing            Boolean  @default(false)
  agreedAt             DateTime
  ipAddress            String?
  createdAt            DateTime @default(now())

  consumer Consumer? @relation(fields: [consumerId], references: [id])
  seller   Seller?   @relation(fields: [sellerId], references: [id])

  @@map("consent_records")
}
```

### Consumer, Seller 모델에 relation 추가

```prisma
model Consumer {
```
